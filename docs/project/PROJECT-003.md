## LLM-ревизия слияния (перспектива)

Слияние онтологий и слияние массивов триплетов — ключевой интеллектуальный и творческий этап; текущая реализация опирается на формальные процедуры (объединение графов, фиксированные правила коллизий). В перспективе предусматривается привлечение **LLM** к этим шагам:

- **Роль LLM:** семантическая проверка на дублирование (не только синтаксическое), осмысленное разрешение коллизий (противоречивые описания, роли, даты) с опорой на исходные тексты и при необходимости на внешние источники; при сомнениях — запрос решения пользователю.
- **Архитектура:** правила остаются первой линией; LLM вызывается для турбулентных/конфликтных случаев; протокол «сомнение → пользователь» (варианты + обоснование); результаты ревизии фиксируются для воспроизводимости.
- **Задачи:** 6.23 (LLM-ревизия слияния онтологий), 6.24 (LLM-ревизия слияния триплетов) в [TASKS.md](../tasks/TASKS.md); детализация — в плане развёртывания (перспектива фазы 4).
